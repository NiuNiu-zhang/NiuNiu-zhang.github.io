Boa服务器和cgi

  • 项目技术栈
  • 2020-08-12

描述：
boa是基于 http 的单任务小型服务器 大小只有 60KB 左右
只能依次 完成用户的请求， 而不会 fork 出新的进程 来处理并发连接请求

CGI 只是一个进程，用来 提供接口 –>自动目录生成 和 自动文件进行拼接
可靠性，可移植性

服务器模型
C-S
client：张三- id1，李四- id2
server：腾讯
1、先登录 输入账号密码 发送服务器验证
2、向 李四 发送消息
封装信息包
<id1 数据 id2>
3、服务器 先查找是否有 李四
然后把 数据包发给 李四
4、李四发给张三
<id2 数据 id1>

类比 ： <boa 服务器的作用>

CGI— 相当于 客户端1
Web—相当于 客户端2
①查找 CGI 查找 Web
②把 CGI 数据提取出来—>交给对应的网页—>显示

boa.conf 保存的配置信息是从哪里查找到 哪里显示
<cgi.bin>
查找：写入配置文件 boa.conf
从 boa.bin 查找

关闭 boa 服务器
先查找 进程号 ps -axj | grep “boa”
kill -9 pid号
注意：
boa 模块在 该项目上层应用中 是基石组成

boa环境搭建和测试

boa 目录搭建完成之后
开始进行 web 服务器的文件根目录创建

cgi:
<CGI—又叫 通用网关接口 >
是 Web 服务器运行时外部程序的规范，按 CGI 编写的程序可以拓展服务器功能
CGI 应用程序能与浏览器进行交互，还可通过数据API与数据库服务器等外部数据源进行通信，
从数据库服务器中获取数据。格式化为HTML文档后，发送给浏览器，
也可以将从浏览器获得的数据放到数据库中。

A-9 —>CGI—>BOA—>HTML
网页发控制表单—–>CGI—>相应的cgi进程控制硬件

c 和 HTML 语言有一个交互
这是c与cgi的数据交互： .c–>,cgi
其实 cgi 就是app
只是cgi提供了 c 和 html 的超文本协议的交互

CGI程序的工作一般流程就是 ①
接收表单数据
进行数据处理，最后根据处理结果生成新的页面 返回给浏览器

表单一般用  POST 方法提交给服务器，由CGI程序获得
    程序根据 元素名字/值 中的元素名字来区分数据，
    完成数据处理后，再读取相应的模板文件，
        根据注释标记和对应的数据填充到 HTML 文本中，生成最后页面返回给浏览器
